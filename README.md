# 📋 Collaborative Specification Form — Demo App

Веб-приложение для **совместного заполнения спецификаций договора** в режиме реального времени.  
Два пользователя (Заказчик и Менеджер) работают в одной форме одновременно, видят изменения друг друга, общаются в чате и взаимодействуют с AI-ассистентом.

---

## 🚀 Быстрый старт (для демо)

### Шаг 1 — Запустить два окна браузера

Откройте **два разных** окна (или вкладки) браузера по одному адресу:

```
http://localhost:5173
```

> ⚠️ Важно: используйте **одинаковый браузер** (Chrome или Edge) — синхронизация работает через `BroadcastChannel` + `localStorage` внутри одного браузера.

---

## 👥 Роли пользователей

### 🔵 Заказчик (окно №1)

**Что делает:**
- Создаёт заявку на разработку спецификации
- Выбирает шаблон предзаполненной формы
- Заполняет форму с 8 шагами
- Общается с менеджером в чате
- Ставит флаги-замечания на поля
- Использует AI-бота (@bot)

### 🟣 Менеджер (окно №2)

**Что делает:**
- Получает уведомление о новой заявке в реальном времени
- Принимает заявку и подключается к сессии
- Видит заполнение формы заказчиком в реальном времени
- Задаёт вопросы, ставит флаги с замечаниями
- Взаимодействует с AI-ботом для проверки формы

---

## 🎬 Сценарий демо (пошагово)

### Часть 1: Настройка ролей

**Окно №1 (Заказчик):**
1. Выбрать роль **«Заказчик»**
2. Нажать на карточку профиля (например, «Алексей Иванов») или ввести своё имя
3. Выбрать цвет → нажать **«Войти как Заказчик»**

**Окно №2 (Менеджер):**
1. Выбрать роль **«Менеджер»**
2. Выбрать другой профиль (например, «Мария Петрова»)
3. Нажать **«Войти как Менеджер»**

---

### Часть 2: Создание заявки (Заказчик)

**В окне Заказчика:**

1. Открылся **Кабинет заказчика**
2. Обратить внимание на блок **«Шаблон формы»** — здесь видны доступные шаблоны:
   - 🟢 «SberNBA — Стандартный шаблон» — с предзаполненными данными
   - ⚪ «Начать с нуля» — пустая форма
3. Выбрать **«SberNBA — Стандартный шаблон»** (нажать на него)
4. В блоке «Детали заявки» ввести название:
   > `Спецификация к договору №5066854`
5. Описание (необязательно):
   > `Трайб SberNBA, разработчик Hadoop, Q3 2025`
6. Нажать **«Создать заявку с шаблоном»**

✅ Форма автоматически заполнится данными из шаблона — видно по подсветке и прогрессу шагов

---

### Часть 3: Менеджер получает уведомление

**Сразу после создания заявки переключитесь в Окно №2 (Менеджер):**

1. В правом верхнем углу появится **тост-уведомление** с анимацией:
   > 🔔 Новая заявка: «Спецификация к договору №5066854»
2. Нажать **«Принять и подключиться»**
3. Менеджер попадает в ту же рабочую сессию

---

### Часть 4: Совместная работа в реальном времени

Теперь оба участника находятся в одной форме.

#### 4.1 Real-time синхронизация полей

**Окно №1 (Заказчик):** Перейти на Шаг 1 («Jira ссылка»), изменить значение поля — например, добавить другой URL.

**Окно №2 (Менеджер):** Видно, что поле обновилось мгновенно! В поле появляется **цветной индикатор** «печатает...» с именем заказчика.

#### 4.2 Навигация по шагам

**Заказчик:** Перейти на Шаг 2 («Трайб и кластеры»).  
**Менеджер:** В боковой панели навигации рядом со Step 2 появляется **аватар заказчика** (обозначает, что он сейчас там).

#### 4.3 Флаги замечаний

**Менеджер** видит поле «Название трайба» и хочет уточнить:
1. Нажать **кнопку 🚩** справа от поля
2. Появится меню — выбрать тип: 🔴 Ошибка / 🟡 Предупреждение / 🔵 Вопрос
3. Ввести комментарий: `Уточните полное название трайба`
4. Нажать «Добавить»

**Заказчик** видит:
- Поле подсвечено флагом
- В топ-баре появился счётчик 🚩1
- В чате появилось сообщение о флаге

#### 4.4 Чат и вопросы

**В боковой панели справа** — чат с реальным временем.

**Менеджер:** ввести в чат:
> `Подтвердите номер договора, пожалуйста`

**Заказчик:** видит сообщение мгновенно, отвечает:
> `Да, договор №5066854 от 01.11.2024`

---

### Часть 5: AI-ассистент (@bot)

Бот всегда в чате и готов помочь.

#### Базовые команды (ввести в чат):

| Команда | Что произойдёт |
|---|---|
| `@bot привет` | Бот поздоровается и покажет список команд |
| `@bot статус` | Бот покажет прогресс заполнения по всем 8 шагам |
| `@bot проверь` | Полный анализ: что заполнено, что пропущено |
| `@bot ошибки` | Поиск логических ошибок (пересечение дат, нулевые суммы) |
| `@bot подскажи` | Подсказка по текущему шагу |
| `@bot флаги` | Список всех активных флагов |
| `@bot помощь` | Полный список команд |

#### Демо-диалог с ботом:

1. **Заказчик** пишет в чат: `@bot проверь`
2. Бот «печатает» (~1-2 сек с анимацией точек)
3. Бот отвечает развёрнутым отчётом по каждому разделу формы

4. **Менеджер** пишет: `@bot ошибки`
5. Бот анализирует даты этапов и суммы

---

### Часть 6: Шаблоны форм (функция Заказчика)

**В рабочей сессии (окно Заказчика):**

1. Нажать кнопку **«Шаблоны»** в верхней панели (рядом с именем пользователя)
2. Открывается **модальное окно управления шаблонами**

#### Загрузить шаблон:
- Вкладка «Мои шаблоны» → карточка шаблона → кнопка **«Загрузить»**
- Форма мгновенно заполняется данными из шаблона
- **Менеджер видит обновление в реальном времени**

#### Сохранить текущую форму как шаблон:
- Вкладка «Сохранить текущую форму»
- Ввести название: `Мой шаблон Q3 2025`
- Добавить описание и теги через запятую: `SberNBA, Hadoop`
- Нажать **«Сохранить как шаблон»**

#### Управление шаблонами:
- Переименование (иконка ✏️ при наведении)
- Удаление с подтверждением
- Счётчик использований

---

### Часть 7: Заполнение формы по шагам

Форма состоит из **8 шагов**. При использовании шаблона «SberNBA» большинство уже заполнено.

| Шаг | Название | Что заполнять |
|---|---|---|
| 1 | Jira ссылка | URL задачи в Jira |
| 2 | Трайб и кластеры | Название трайба, продукты и коды |
| 3 | Ответственные | ФИО ответственных, подрядчик, НФТ |
| 4 | Договор | Номер и дата договора |
| 5 | Предметы спецификации | Предмет договора и CI |
| 6 | Этапы | Этапы работ, сроки, стоимость |
| 7 | Функциональные требования | ФТ по каждому этапу |
| 8 | Сотрудники | ФИО, роли, ставки, часы |

#### Индикация прогресса:
- ✅ Зелёный — шаг завершён
- 🔵 Синий — текущий шаг  
- ⚪ Серый — не заполнен
- 🔴 Красный — есть флаги замечаний

## 🔧 Ключевые фичи для показа

### ✅ Real-time синхронизация
Все изменения формы видны второму участнику без перезагрузки страницы.

### ✅ Индикаторы присутствия
- Аватары пользователей в навигации по шагам (кто где находится)
- «Печатает...» при вводе текста в поле
- Онлайн-индикатор в топ-баре

### ✅ Система флагов
- 3 типа: 🔴 Ошибка, 🟡 Предупреждение, 🔵 Вопрос
- Флаги видны обоим участникам
- Можно закрыть (снять) флаг после исправления
- Флаги дублируются в чате

### ✅ Чат в реальном времени
- Сообщения с именами и цветами участников
- Системные сообщения (подключение/отключение)
- AI-бот с командами

### ✅ AI-ассистент
- Понимает контекст формы
- Даёт рекомендации по заполнению
- Находит логические ошибки
- Отвечает через ~1-2 секунды (с анимацией)

### ✅ Шаблоны форм (Заказчик)
- Предзаполненные демо-шаблоны
- Сохранение текущей формы как шаблон
- Применение шаблона при создании заявки
- Управление: переименование, теги, удаление

### ✅ Система ролей
- Заказчик → создаёт заявку → выбирает шаблон → заполняет форму
- Менеджер → получает уведомление → принимает → совместная работа

## 🔄 Как работает синхронизация

```
Вкладка 1 (Заказчик)           Вкладка 2 (Менеджер)
      │                               │
      │ ── изменил поле ──────────>   │
      │                    localStorage (SharedState)
      │ <───────── BroadcastChannel ──│
      │      (событие: FORM_UPDATE)   │
      │                               │
      │     setSharedState({...})     │
      │         re-render             │
```

- Все изменения сохраняются в `localStorage`
- `BroadcastChannel` уведомляет другие вкладки об изменениях
- Вкладка-получатель читает актуальное состояние и обновляет UI
